## Add Clear History + PDF Export to Likeness Monitor

Two additions to the **Run a Scan** page (`/dashboard/monitor`) so performers can reset their scan list after taking action and download a clean report.

### 1. Clear History controls (`src/components/likeness/ScanHistory.tsx`)
- Header row gets two buttons next to "Scan History":
  - **Download PDF Report** (outline)
  - **Clear All History** (destructive, with confirm dialog)
- Per-scan: a small trash icon to delete just that single scan.
- "Clear All" shows an `AlertDialog` ("This will delete all X scans. You can re-scan anytime.") — on confirm, deletes all rows in `likeness_scans` for the current user, then refreshes.
- After clearing, the empty state nudges: "History cleared. Run a new scan above to see current results."

### 2. PDF Report Generator (`src/lib/likenessReport.ts` — new)
- Uses `jspdf` + `jspdf-autotable` (already common; will add via `bun add` if missing).
- Pulls the user's `legal_name` and `full_name` (stage name) from the `profiles` table.
- Layout:
  - **Header**: ClaimMyFace crimson bar with logo text, "Likeness Scan Report"
  - **Performer block**: Legal Name, Stage Name, Email, Report generated date
  - **Summary**: total scans, total findings, breakdown by scan type
  - **Per-scan tables**: query, date, type, then a table of results (Title | URL | Match type | Description)
  - **Footer**: "Confidential — generated by ClaimMyFace.com" + page numbers
- Saves as `ClaimMyFace-Scan-Report-{stageName}-{YYYY-MM-DD}.pdf`.

### 3. Wire-up (`src/pages/LikenessMonitor.tsx`)
- Pass `onClearAll` and `profile` (fetched once on mount) into `ScanHistory`.

### Files
- **Create**: `src/lib/likenessReport.ts`
- **Edit**: `src/components/likeness/ScanHistory.tsx`, `src/pages/LikenessMonitor.tsx`
- **Add deps**: `jspdf`, `jspdf-autotable` (if not present)

No backend/schema changes — uses existing `likeness_scans` and `profiles` tables.
